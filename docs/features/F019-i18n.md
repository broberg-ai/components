# F019 — i18n / Language Switch

> L2 Shell · hybrid · effort **M** · impact **medium** · owner `trail`. Status: Backlog.
> Graduate-candidate: no — stays in `components`.

## Motivation
A headless locale engine + thin stack adapters handling: locale detection (localStorage, navigator.language), persistence (localStorage + cookie for SSR), dot-path string lookup with fallback chain, interpolation ({var}), and a bilingual() helper for LLM-generated bilingual content objects. The shell adapter exposes a LanguageSwitcher that switches locale by URL-prefix replace (Next App Router) or in-memory pub/sub (SPA). The core contract is already stable + nearly identical across trail, sanneandersen, xrt81, cms-docs.

## Solution
**hybrid.** The headless core (Locale type, SUPPORTED_LOCALES, detectInitial, set/getLocale, pub/sub, t() lookup, bilingual()) is identical across trail i18n.ts, sanneandersen i18n.ts, xrt81 i18n.tsx — three production repos with the same contract, diverging only in storage-key naming + the BilingualText import → runtime-package engine. The UI widget differs per stack (cms-docs slug-suffix URL rewrite, sanneandersen /[locale] prefix + cookie, xrt81 in-memory context) → copy-owned. So: shared engine + copy-owned adapter UI.

## Architecture

### Best source (reference implementation)
`broberg/trail` — `apps/admin/src/lib/i18n.ts` + `locales/{en,da}.json`: detectInitial (localStorage + navigator.language), global pub/sub set/getLocale without a context provider, useLocale(), t() dot-path + {var} interpolation + en fallback + raw-key last-resort, bilingual() string-tolerant for legacy plain strings, html[lang] sync, no next/* imports. Comment documents the extensibility contract (add JSON + list in SUPPORTED_LOCALES).

### Other implementations seen
- `webhouse/sanneandersen` `site/src/lib/i18n.ts` + `app/[locale]/layout.tsx` + `components/layout/top-bar.tsx` — best Stack A: [locale] dynamic segment, generateStaticParams, switchLocale() cookie + router.push, generateMetadata hreflang.
- `broberg/xrt81` `apps/web/src/lib/i18n.tsx` — cleanest Stack B context: LangProvider + useLang + tr<T>(lang,{da,en}) typed inline pairs (no dict file).
- `webhouse/cms-docs` `src/components/locale-switcher.tsx` — only slug-suffix routing (/-da); most polished dropdown (flag + label + checkmark).

### Headless core vs. adapters
- **Core (no React/next):** Locale; LocaleDef {code,label,flag?}; createI18n({locales, defaultLocale, storageKey?, dicts?}) → getLocale/setLocale/subscribe/t(key,vars?)/bilingual(text)/detectInitial; t() = dot-path → locale-then-en → raw-key + {var}; bilingual() picks {en,da} field or returns string. Configurable storageKey.
- **Stack A (Next/React/shadcn):** useLocale() (pub/sub via useState+useEffect, no context); LanguageSwitcher (usePathname+useRouter URL-prefix swap + locale cookie, sanneandersen pattern); generateI18nStaticParams(); shadcn DropdownMenu variant. No next/* in core.
- **Stack B (Bun/Hono/Preact):** useLocale() (preact/hooks over core pub/sub); LangProvider context variant (xrt81); Preact LanguageSwitcher (pure setLocale, no router); optional Hono localeMiddleware() reading Accept-Language + cookie.

### Public API
```ts
export function createI18n(config: { locales: LocaleDef[]; defaultLocale: string; storageKey?: string; dicts?: Record<string,Record<string,unknown>> }): { getLocale; setLocale; subscribe; t; bilingual; detectInitial };
// '@broberg/i18n/react' → useLocale, LanguageSwitcher, generateI18nStaticParams
// '@broberg/i18n/preact' → useLocale, LangProvider, LanguageSwitcher
```

## Risks
Module-level singleton (currentLocale + listeners Set) breaks with multiple loaded instances (pnpm hoist) — use a globalThis-keyed guard. BilingualText currently imports from @trail/shared — extraction must make it a generic Record<string,string>. cookie-name collision (sanneandersen 'sa-locale' vs others) — configurable cookieName resolves it. Keep dicts in-repo; package provides only the engine.
